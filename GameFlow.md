# Atividade de Desenvolvimento: Jogo no Terminal

## Objetivo

O objetivo desta atividade é o desenvolvimento de um jogo no terminal usando a biblioteca `readline-sync` em **JavaScript** ou **Typescript**. O jogo a ser desenvolvido será definido pela dupla de alunos, sendo livre, desde que rodem corretamente no terminal e utilizem a biblioteca mencionada para interação com o usuário.

Além disso, a atividade tem o objetivo de trabalhar o uso de **GitFlow** no desenvolvimento de projetos colaborativos, com foco no uso de **commits semânticos** e na **resolução de conflitos** de maneira eficaz, caso ocorram durante o desenvolvimento.

## Instruções

1. **Desenvolvimento do Jogo:**
   - O jogo deve ser desenvolvido em **JavaScript**, utilizando a biblioteca `readline-sync` para capturar entradas do usuário.
   - O enredo e regras do jogo são definidos pela dupla. Pode ser um jogo simples como um "Jogo da Forca", "Adivinhação", "Quiz", entre outros.
   - O jogo deve ser interativo e funcional no terminal, com o usuário podendo jogar até o fim.

2. **Utilização do GitFlow:**
   - Criação e organização do repositório de acordo com a metodologia **GitFlow**.
   - Utilizar commits semânticos como parte do fluxo de desenvolvimento. Seguir as convenções:
     - `feat`: Adiciona uma nova funcionalidade.
     - `fix`: Corrige um bug.
     - `docs`: Atualizações ou adições à documentação.
     - `style`: Ajustes de estilo (formatação, indentação, etc).
     - `refactor`: Refatoração de código.
   - Durante o desenvolvimento, se houverem conflitos no repositório, a dupla deve resolvê-los utilizando boas práticas de **resolução de conflitos**.

3. **README:**
   - O repositório deve conter um arquivo `README.md` básico, contendo:
     - Descrição do jogo.
     - Como executar o jogo no terminal.
     - Dependências necessárias (por exemplo, como instalar o `readline-sync`).
     - Qualquer outra informação importante sobre o funcionamento do jogo.

4. **Entrega:**
   - O repositório deve ser compartilhado no GitHub e a dupla deve enviar o link para o repositório para o email **ledsouza@senacrs.com.br** com o assunto **GameFlow** e com os nomes completos da dupla no corpo do email, junto com o link do repositório.
   - Certifique-se de que todas as funcionalidades do jogo estejam implementadas e que o jogo esteja funcional no terminal.
   - A documentação (README) deve ser clara e objetiva.

## Critérios de Avaliação

- Funcionamento do jogo no terminal.
- Organização do código.
- Aplicação correta do GitFlow e commits semânticos.
- Qualidade da resolução de conflitos (se houver).
- Clareza e objetividade do README.
